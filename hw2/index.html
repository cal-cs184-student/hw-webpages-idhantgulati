Overview

This homework covers two topics in geometric modeling. The first section builds up Bezier curves and surfaces using de Casteljau subdivision, starting with 1D recursive linear interpolation and extending it to a separable two-pass algorithm for evaluating points on Bezier surfaces. The second section dives into the half-edge mesh data structure, implementing area-weighted vertex normals for smooth shading, edge flips and splits for local mesh editing, and Loop subdivision for mesh upsampling. Bonuses include boundary edge splits, boundary-aware Loop subdivision, and a modified Butterfly subdivision scheme.

§ 1

Bezier Curves with 1D de Casteljau Subdivision

Algorithm Overview

De Casteljau's algorithm is a recursive method for evaluating a point on a Bezier curve at a given parameter \(t \in [0, 1]\).

Given \(n\) control points \(p_1, p_2, \ldots, p_n\), each subdivision step linearly interpolates between consecutive pairs of points to produce \(n-1\) new intermediate points:

\[p_i' = (1 - t)\,p_i + t\,p_{i+1}\]

We repeat this step on the resulting points until only a single point remains. That final point lies on the Bezier curve at parameter \(t\).

Implementation

In BezierCurve::evaluateStep(...), I iterate over consecutive pairs of the input points and compute the linear interpolation using the member variable t:

std::vector<Vector2D> BezierCurve::evaluateStep(std::vector<Vector2D> const &points) {
    std::vector<Vector2D> result;
    for (int i = 0; i < (int)points.size() - 1; i++) {
        result.push_back((1 - t) * points[i] + t * points[i + 1]);
    }
    return result;
}

Each call reduces the point count by one. Calling this repeatedly on a curve with \(n+1\) control points eventually yields the single evaluated point on the curve.

§ 2

Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

Implementation

To compute a smooth per-vertex normal, I walk all faces incident to the vertex using the half-edge structure. For each face, I grab its three vertex positions via the half-edge chain and compute the cross product of two edge vectors:

\[\vec{n}_{face} = (\vec{p_1} - \vec{p_0}) \times (\vec{p_2} - \vec{p_0})\]

The magnitude of this cross product is \(2 \times \text{area}\), so it naturally weights the face's normal contribution by area, no separate area calculation needed. I skip boundary half-edges (virtual boundary faces should not contribute). After accumulating all face contributions, I normalize the result to get a unit normal.

Part 4: Edge Flip

Implementation

An edge flip takes the shared edge \((b,c)\) between triangles \((a,b,c)\) and \((c,b,d)\) and re-connects it as edge \((a,d)\), yielding triangles \((a,d,c)\) and \((d,a,b)\). No elements are created or destroyed, only pointers are rewired.

I first label all 6 half-edges, 4 vertices, and 2 faces by walking the half-edge chain from e0->halfedge():

h0 = e0->halfedge()          // b→c  (face f0)
h1 = h0->next()              // c→a
h2 = h1->next()              // a→b
h3 = h0->twin()              // c→b  (face f1)
h4 = h3->next()              // b→d
h5 = h4->next()              // d→c

After the flip, h0 becomes \(a \to d\) and h3 becomes \(d \to a\):

f0: (a,d,c)  →  h0(a→d), h5(d→c), h1(c→a)
f1: (d,a,b)  →  h3(d→a), h2(a→b), h4(b→d)

I use setNeighbors to set all 5 fields of every half-edge at once, then update the halfedge pointer for each vertex and face. Boundary edges are rejected immediately.

Debugging

Drawing the before/after diagram on paper and labeling every half-edge before writing any code was essential. Setting all pointers, even unchanged ones, avoided subtle bugs from stale pointers.

Part 5: Edge Split

Implementation

A split inserts a new midpoint vertex \(m\) on edge \((b,c)\) and connects it to the opposing vertices, turning 2 triangles into 4. This creates new mesh elements: 1 vertex, 3 edges, 2 faces, and 6 half-edges.

New vertex position:

\[m.\text{position} = \frac{b.\text{position} + c.\text{position}}{2}\]

New triangles (reusing existing half-edges where possible):

f0: (a, b, m)  →  h2(a→b),   h0(b→m),   h_ma(m→a)
f2: (m, c, a)  →  h_mc(m→c), h1(c→a),   h_am(a→m)
f1: (m, b, d)  →  h3(m→b),   h4(b→d),   h_dm(d→m)
f3: (c, m, d)  →  h_cm(c→m), h_md(m→d), h5(d→c)

The original edge e0 is reused for the \(b \to m\) / \(m \to b\) half-edges. The 3 new edges are \(e_{mc}\), \(e_{ma}\), and \(e_{md}\).

Boundary Edge Splits [bonus]

For boundary edges, only one side has a real face. The implementation orients h0 to always point into the real face first:

if (h0->face()->isBoundary()) h0 = h0->twin();

Then it creates only 2 real triangles from the one real face, and splices the new boundary half-edges into the existing boundary loop. The virtual boundary face stays a single polygon, it just gains one extra vertex.

Debugging

m->halfedge() must point along the original split edge direction, so it is set to h3 (\(m \to b\)).

For the boundary case, a small loop walks the boundary polygon to find the predecessor of h3 so the new h_cm can be correctly spliced in.

One subtle bug: vertex \(d\) must be read from h5->vertex() (since h5 is \(d \to c\)), not h4->vertex() (which is \(b\), the source of \(b \to d\)). Getting this wrong silently connected the wrong vertex, causing corrupted geometry that was hard to spot until edges were flipped on the split mesh.

Part 6: Loop Subdivision for Mesh Upsampling

Implementation

Loop subdivision is done in four steps, computing positions before subdividing:

Step A - Precompute new positions on the original mesh:

For each old vertex with degree \(n\):

\[\text{newPosition} = (1 - n \cdot u) \cdot \text{pos} + u \cdot \sum_{\text{neighbors}} \text{pos}_i\]

where \(u = \frac{3}{16}\) if \(n = 3\), else \(u = \frac{3}{8n}\).

For each edge, the new midpoint position:

\[\text{newPosition} = \frac{3}{8}(A + B) + \frac{1}{8}(C + D)\]

where \(A, B\) are the edge endpoints and \(C, D\) are the opposite vertices of the two adjacent triangles. All stored in newPosition without modifying position, since the averages must use original positions.

Boundary Mesh Support [bonus]

For boundary edges, the midpoint position is simply \(\frac{A+B}{2}\). Boundary vertices keep their original positions unchanged. This reuses the boundary edge split support from Part 5.

Step B - Split every original edge:

All original edges are collected into a list first to avoid splitting newly created edges. Each split midpoint's newPosition is set from the edge's precomputed value.

Step C - Flip new edges connecting an old and a new vertex:

Each new edge (isNew) is flipped if its two endpoints differ in isNew status, completing the 4-1 subdivision pattern.

Step D - Apply positions:

Every vertex's position is set to its newPosition.

Observations on Sharp Features

After loop subdivision, sharp corners and edges get smoothed out as the weighted average pulls vertices toward their neighbors. Pre-splitting edges along a sharp crease before subdividing partially preserves the feature.

Cube Symmetry

Subdividing dae/cube.dae repeatedly produces an asymmetric result. Each cube face has a single diagonal, and these diagonals are not arranged symmetrically across all 6 faces. Since Loop weights depend on vertex degree, asymmetric connectivity causes vertices to be pulled unevenly.

Fix: Pre-process each face to have an X-pattern (split each face diagonal). This gives every corner vertex the same degree and neighborhood structure, so subdivision is uniform and the cube stays symmetric.

Modified Butterfly Subdivision [bonus]

I implemented the modified Butterfly scheme (butterfly_upsample). Unlike Loop, Butterfly is interpolating, old vertices keep their positions exactly, so the subdivided mesh passes through the original vertices.

The new midpoint uses an 8-point stencil when both edge endpoints are regular (degree 6) and no adjacent face is on the boundary:

\[m = \frac{1}{2}(A+B) + \frac{1}{8}(C+D) - \frac{1}{16}(e_0 + e_1 + f_0 + f_1)\]

where \(C, D\) are the immediate wing vertices and \(e_0, e_1, f_0, f_1\) are the outer-ring vertices. For irregular vertices or near boundaries, a 4-point fallback is used:

\[m = \frac{3}{8}(A+B) + \frac{1}{8}(C+D)\]

The split/flip/apply steps are identical to Loop subdivision.
